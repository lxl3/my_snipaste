# 文件处理与保存功能说明

## 📁 自动保存功能

MySnipaste 提供灵活的自动保存功能，可以在截图完成后自动保存到指定目录。

### 配置自动保存

1. 打开设置对话框（托盘图标右键 → 设置，或使用快捷键 Ctrl+，）
2. 切换到 "常规" 选项卡
3. 勾选 "自动保存到目录"
4. 点击 "浏览..." 选择保存目录
5. 选择保存格式（PNG 或 JPEG）
6. 点击 "保存" 应用更改

### 文件命名规则

自动保存的文件名按照以下格式生成：

```
Screenshot_{时间戳}.{格式}
```

例如：`Screenshot_20260524_143022.png`

其中：
- 时间戳格式：`YYYYMMDD_HHMMSS`（年月日_时分秒）
- 格式：根据设置选择的格式（png 或 jpeg）

### 手动保存

除了自动保存外，用户还可以通过以下方式手动保存截图：

1. 在标注完成后，点击工具栏中的保存按钮（磁盘图标）
2. 使用快捷键 Ctrl+S
3. 在标注完成后，按回车键（如果未开始文字编辑）

手动保存将打开文件保存对话框，允许用户：
- 选择保存位置
- 修改文件名
- 选择文件格式（PNG、JPEG 或所有文件）

## 💾 支持的文件格式

MySnipaste 目前支持以下图像格式的保存：

| 格式 | 文件扩展名 | 说明 |
|------|------------|------|
| PNG | .png | 无损压缩，支持透明度，推荐用于截图 |
| JPEG | .jpg/.jpeg | 有损压缩，文件较小，适合照片 |
| 所有文件 | * | 自定义扩展名（实际保存为PNG） |

### 格式选择建议

- **PNG**：推荐用于大多数截图场景，特别是包含文字、线条或需要透明背景的图像
- **JPEG**：适合于包含大量渐变色彩的图像（如照片），可以显著减小文件大小
- **所有文件**：允许指定自定义扩展名，但实际仍以PNG格式保存

## 📂 默认保存行为

当未配置自动保存目录时，MySnipaste 的默认行为是：

1. 手动保存时：使用上次手动保存的目录（如果有），否则使用用户的主目录
2. 自动保存时：不执行任何保存操作（仅在配置了目录时生效）
3. 复制到剪贴板：始终可用，不受保存设置影响

## 🔧 技术实现细节

### 文件对话框实现

保存功能使用 Qt 的 `QFileDialog.getSaveFileName()` 方法实现：

```python
file_path, _ = QFileDialog.getSaveFileName(
    None,                           # 父窗口（None 表示无父窗口）
    _("Save Screenshot"),          # 对话框标题
    default_path,                   # 默认路径和文件名
    _("PNG Image (*.png);;JPEG Image (*.jpg *.jpeg);;All Files (*)"),  # 文件过滤器
)
```

### 路径处理

1. **默认路径计算**：
   - 如果配置了自动保存目录：`目录/截图_{时间}.{格式}`
   - 如果未配置自动保存目录：仅文件名 `截图_{时间}.{格式}`

2. **目录创建**：
   - 自动保存时，如果目录不存在，会尝试创建它
   - 如果创建失败（权限不足等），会回退到仅使用文件名模式

3. **文件名生成**：
   - 使用当前时间戳确保文件名唯一
   - 时间格式遵循 `YYYYMMDD_HHMMSS` 便于排序

### 保存过程

当用户选择保存路径后：

1. 验证所选路径的有效性
2. 根据文件扩展名确定实际保存格式（强制小写）
3. 使用 `QPixmap.save()` 方法保存图像
4. 保存成功后自动关闭截图覆盖层（如果是通过标注工具保存的）

## ⚠️ 常见问题及解决方案

### 问题：保存失败或没有反应

**可能原因**：
- 目录不存在且无权限创建
- 磁盘空间不足
- 文件名包含非法字符
- 目录路径过长

**解决方案**：
1. 检查所选目录是否存在且可写
2. 确保磁盘有足够的空间
3. 避免在文件名中使用特殊字符如：`\/:*?"<>|`
4. 选择较短的路径或更上层的目录

### 问题：自动保存没有工作

**可能原因**：
- 未在设置中启用自动保存
- 未选择有效的保存目录
- 目录路径错误或不可访问

**解决方案**：
1. 打开设置确认“自动保存到目录”已勾选
2. 确认已选择有效的目录路径
3. 尝试重新选择目录或检查目录权限

### 问题：保存的图片质量不佳

**可能原因**：
- 选择了JPEG格式且质量设置较低
- 原始截图分辨率较低
- 图像在保存过程中被无意压缩

**解决方案**：
1. 对于需要高质量的图像，请使用PNG格式
2. 确保在高分辨率显示器上进行截图
3. 检查是否有其他软件在后台处理了图像

## 🎯 最佳实践

### 截图保存建议

1. **重要截图**：使用PNG格式并启用自动保存到专用目录
2. **临时截图**：仅使用复制到剪贴板功能（Ctrl+C）
3. **需要共享的图像**：根据目标平台选择PNG或JPEG
4. **定期整理**：定期清理旧的截图文件以节省磁盘空间

### 性能优化

1. 自动保存目录建议使用SSD而非网络驱动器
2. 对于非常大的截图（4K+），考虑先标注再保存以减少处理时间
3. 如果经常进行截图，考虑将自动保存目录设置为桌面或文档文件夹以便快速访问

### 安全考虑

1. 避免将敏感信息的截图保存到云同步目录（如Dropbox、iCloud）中，除非已加密
2. 定期检查自动保存目录，及时删除不需要的截图
3. 使用具有访问控制的目录来保存包含个人信息的截图
